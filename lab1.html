¿Cuál es el propósi to de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Los metodos sirven para indicar que accion va realizar.

GET - Es el método que solicita un recurso, usualmente se usan para recuperar datos.

HEAD - Es el método que pide la respuesta identica de la petición GET

POST - Es el método que envia una entidad a un recurso

PUT - R

PATCH - Borra un recurso especifico

DELETE - Borra un recurso especifico

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que la solicitud tuvo éxito, esto depende del método http que se haya usado.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No necesariamnete significa que es un error que se debe al equipo de desarrollo. El error 404 se debe cuando el usuario ha escrito mal el URL.

Sin embargo, otras causas pueden ser que se cambio la estructura de la URL en la página o se haya elminado dicha página.
